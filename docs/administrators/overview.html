 Features

 

    Personalization

  • Choose color scheme
  • Choose channels
  • Arrange channels in a tab/column layout

    • Publish

  • Make a channel available for subscription

    • Subscribe

  • Browse and choose channels which are grouped by category
  • Preview a channel before subscribing

    • Included channels

  • CApplet: Displays a java applet
  • CAuthorization: Lets users log on
  • CBookmarks: Lets users store their favorite web sites
  • CInlineFrame: Displays an html page
  • CPageRenderer: Displays an html page
  • CRSSChannel: Displays an RSS channel
  • CSearch: Provides a form for searching the web

 Requirements

 
  • A Java Servlet Engine/Container which is Servlet 2.2/JSP 1.1 compliant. uPortal has been tested under Tomcat and Caucho's Resin
  • JDK 1.2.2 or higher Java Virtual Machine. uPortal has been tested on the SUN Virtual Machine, but not the Microsoft VM.
  • Netscape Navigator 4.x or Internet Explorer 4.x as a minimum. Mozilla or Internet Explorer 5.x should also work fine.
  • The ObjectSpace DXML classes. uPortal requires these classes to process XML.
  • IBM's XML Parser For Java. DXML requires these classes for XML parsing. The correct version of XML4J can be downloaded from ObjectSpace here.
  • A JDBC compliant database. uPortal is developed using HypersonicSQL. Oracle has also been used. Any JDBC compliant database should work with some modification to the CREATE scripts and some SQL statements.
  • An XSLT engine. uPortal used Apache's Xalan.

 Installation

 
  • uPortal is 100% Java and should therefore run on any platform.
  • Move the "webpages" directory to "/webpages" under your web server. If you already have a "webpages" directory simply add the files there.
  • Put all required class packages along with the uPortal classes in the classpath of the servlet engine.
  • Run Hypersonic (hsql) in client/server mode by initiating the RunWebServer script. After starting hsql, run the DatabaseManager applet and log into hsql as client/server using the default login. Load the scripts from the 'scripts' directory. Execute the scripts to create and populate the tables.
  • Set the portal base directory in the web.xml file. Here is an example: 
          <web-app>
        <context-param>
          <param-name>portalBaseDir</param-name>
          <param-value>D:\Projects\JA-SIG\portal\</param-value>
        </context-param>
      </web-app>
  • Start the servlet engine and load index.jsp in a web browser.

 CVS Access

 

    All Users

  • All users wishing to contribute to the project must first register with the JA-SIG Clearinghouse. The login and password you set here will be the login and password you will use for accessing CVS.
  • We have created a generic account on our server for logging in and creating the SSH tunnel. The instructions for logging into this account will be sent you upon request.

    • CVS Access on Windows

  • On Windows, you need to use a CVS client and SSH client/tunnel.

    • WinCVS 1.1b7 or greater (GUI client)
    • MindBright MindTerm Java SSH

    • You can find other SSH clients here but the above are the only ones recommended since they do SSH tunneling (most SSH clients just do terminal emulation).

    In addition, if you are behind a corporate firewall, your best option is to use a socks proxy. Most SSH software knows how to traverse socks proxies. In addition, there are socks utility programs that let you "sockisfy" any application. Contact your sysadmin for details.

  • How to Connect using MindTerm and WinCVS

    This is an illustrated example showing the MindTerm program and the WinCVS CVS GUI client.

    Assuming you have installed the Mindterm classes (or .jar archive) on your system classpath (and, of course, that you have a Java virtual machine available on your system), the first step is turn on the client. From the DOS prompt type:

    java mindbright.application.MindTerm

    Now, go to the tunnels menu and set is up so that local port 2401 is directed to host "localhost" on the same (you don't need to understand why) as showed in the picture below:

    When this is done, go ahead and connect to SSH host mis105.mis.udel.edu by typing so at the SSH-server: prompt. Then your username will be requested and the password. Don't worry about transmitting your password since SSH is done exactly to allow you to transmit your password without worrying about sniffing and other security holes of normal remote shells.

    When mis105 welcomes you and its prompt appears, you can safely minimize your mindterm window since, for plain WinCVS operation, you won't need to type anything in the ssh terminal. The WinCVS client will send CVS commands thru the SSH tunnel transparently.

    Then launch the WinCVS client. The first time, it will ask you the basic configuration preferences that it needs. These are the CVS repository location and authentication information. The screenshot below tells you what you should enter. Note that you must replace the string username in the repository location with your own username on mis105.

    At this point you software is setup and first thing to do is to authenticate yourself on the CVS repository. This is done by logging in the server with the menu Admin -> Login and inserting your password. This action authenticates you into the CVS server and saves your password (in a crypted form) in the .cvspass file in the home directory you specify. Note that you need to do this only once per CVS repository, not everytime you connect to it, even if you disconnected from that server.

    Now your WinCVS client is ready to perform operations on the remote CVS repository. First action to take is to checkout the portal module. To do this, first select the directory where you want the files to be located. Note that the files will be automatically placed in the portal subdirectory of the indicated directory. Then select the menu item Create -> Checkout module.... After this, place portal in the module name field and you're ready to go.

    • CVS Access on Unix

  • On Unix, you need the following software products:

    • CVS client (pick up latest version)
    • SSH client (pick up 1.x, not 2.x)

    Depending on your platform you may be able to pick up precompiled versions or have to compile them yourself.

  • Notes

    Since both CVS and SSH are based on open standards, there are lots of CVS and SSH tools freely available on the net, both of command line and GUI behavior. This means that any compliant implementations of those standards should work as well.

    For some more software options as well as other tips and tricks, see www.hyperreal.org/info/ssh

    In addition, if you are behind a corporate firewall, your best option is to use a socks proxy. Most SSH software knows how to traverse socks proxies. In addition, there are socks utility programs that let you "sockisfy" any application. Contact your sysadmin for details.

  • How to Connect

    The first step is to set up the SSH tunnel to mis105. To do this, you need to have your SSH client software redirect your local port 2401 to the far side of the SSH connection. The command looks something like this:

    ssh -L 2401:localhost:2401 mis105.mis.udel.edu

    The next step is to use the CVS pserver functionality to login. You'll want to do this in a seperate terminal from the one that you logged into mis105 with so that you check out the files to your local machine and not to your account on mis105! This command looks like this:

    cvs -d :pserver:user@localhost:/home/cvs/jasig login

    Note that the CVS client thinks it is talking to a server running on your local machine, but in fact is talking to the CVS pserver on mis105 thanks to the SSH tunnel. After you login, you can check out any of the source trees. The command to checkout the JA-SIG Portal source is:

    cvs -d :pserver:user@localhost:/home/cvs/jasig checkout portal

    Once you checkout a source tree, all CVS operations in that source tree do not require the -d :pserver: argument.
