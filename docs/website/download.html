The uPortal builds provide a "drop it in and go" configuration of uPortal.  The intent is to make it easier for potential content developers to quickly get uPortal up and running and allow them to begin building content. The builds include a working configuration of the Tomcat Servlet Engine and HypersonicSQL database.

Users wanting to setup and demonstrate a uPortal site may download the tar or zip of uPortal 1.5 "Destin".  Content created for the "Destin" release can be ported to uPortal 2 as long as certain programming considerations are adhered to.  To get started with the 2.0 architecture and channel interfaces download the 2.0 "Feature Freeze Release".  See Getting Involved and the Documentation sections of this site for more information about creating content for uPortal.

The source code is included with each distribution, however for access to the latest code, visit the Source Code Repository for source updates.

The uPortal developers at Yale have put these builds together and you can download them using the links below.
 

Release Status Description
2.0 "pre-release" Feature Freeze Release Release incorporates exactly the same set of features as the uPortal 2.0 final release will. Between now and the final release, the development team will fix bugs, clean up the code and work on the documentation. Some minor architecture or database changes might occur throughout the cleanup/bug fix process.
2.0 "LaJolla" Beta Release Release incorporates a number of fixes and improvements over the Sonora alpha release. Most notably normalized database implementation, tab-column specific user preferences module and profile management screens.
2.0 "Sonora" Alpha Release New architecture which makes use of XSLT. Developers will be able to design XSLT stylesheets to render portal on various devices such as web browsers, PDAs, and cell phones.
1.5 "Destin" Stable - Current Includes all the features of 1.0 plus support for authorization (roles)
1.0 "Monterey" Stable - Old Initial release of uPortal.

So what about Plymouth?  The "Plymouth" distribution released around Thanksgiving 2000, was an interim build created between uPortal 1.0 (Monterey) and uPortal 1.5 (Destin).  It was never really intended for general release.
 

And Arvon? Arvon was a pre-release version of the 2.0 architecture and has been superceded by Sonora.
